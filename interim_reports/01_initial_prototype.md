# Interim Report 01: 초기 프로토타입 개발 완료

## 📝 개요
광양고등학교 자공고 2.0 프로그램 안내 및 개인화된 이수 트랙 추천을 위한 웹 어플리케이션의 초기 프로토타입을 구축하였습니다.

## 🛠️ 수행 작업
1.  **Firebase 백엔드 아키텍처 구축**:
    *   `firebase.json`: Hosting 및 Cloud Functions 설정.
    *   `functions/index.js`: Gemini API 호출을 서버 측에서 처리하도록 전환하여 API 키 노출 원천 차단.
2.  **보안 인증 시스템 구현**:
    *   `secondary_data/student_list.json`: 학번+이름 기반의 실제 학생 인증 DB 구조 마련.
    *   `certification/teacher.csv`: 교사용 마스터 코드를 CSV 파일로 관리하여 동적으로 로드.
    *   **학번 입력 유연성 개선**: 4자리(1101) 또는 5자리(10101) 학번 모두 허용하도록 정규화 로직 추가.
3.  **UI/UX 고도화**:
    *   인증 실패 시 에러 메시지 처리 및 피드백 강화.
    *   내신 집중형/탐구 몰입형 선택에 따른 가이드 텍스트 최적화.
    *   네비게이션 바 추가 및 전반적인 레이아웃 정교화.
4.  **데이터 무결성 검증**:
    *   CSV 파싱 및 JSON 연동을 통해 AI가 항상 최신 학교 데이터를 참조하도록 로직 개선.

## 📊 현재 상태
*   로컬 환경에서 학생 인증 및 UI 전환 테스트 완료.
*   백엔드 함수 구조 준비 완료 (Gemini API 서버측 호출).
*   이미지 저장 기능 배율 최적화 (2배수 고화질).
*   **인증 시스템 개선**: 교사 비밀번호 CSV 기반 관리, 학번 4/5자리 모두 허용.
*   **환경 변수 관리**: `.env` 파일로 API 키 관리, 하드코딩 제거.
*   **개발/프로덕션 환경 분리**: 
    - 개발 환경: 프론트엔드에서 직접 Gemini API 호출 (빠른 테스트)
